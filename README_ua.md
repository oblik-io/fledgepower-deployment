# fledgepower-deployment

Містить усі необхідні компоненти для розгортання служб FledgePower.

## розгортання за допомогою all-plugins-ubuntu2004

### Динамічна генерація контейнерів з конфігурації YAML

Цей скрипт оболонки (`buildContainerfile.sh`) призначений для створення Docker-файлу на основі конфігурацій, наданих у YAML-файлі (`buildContainerfile.yml`). Нижче описано, що робить скрипт:

1. Він встановлює змінні для докер-файлу (`dockerfile`) і його шаблону (`dockerfiletemplate`), а також для YAML-файлу (`yaml_file`).
2. Копіює вміст шаблону докерфайлу до докерфайлу.
3. Читає різні значення конфігурації з YAML-файлу за допомогою `yq` (інструмент для обробки YAML-файлів).
4. Оновлює аргументи в докер-файлі (`$dockerfile`) значеннями, прочитаними з YAML-файлу за допомогою `ed`.
5. Встановлює маркер у докер-файлі, куди будуть вставлятися модулі.
6. Отримує список назв та версій модулів з YAML-файлу за допомогою `yq`.
7. Перебирає список модулів, генеруючи Docker-інструкції для кожного модуля і вставляючи їх у Docker-файл у вказаний маркер за допомогою `ed`.
8. Читає значення портів з YAML-файлу.
9. Оновлює інструкцію `EXPOSE` в Docker-файлі портами, прочитаними з YAML-файлу за допомогою `sed`.
   Таким чином, скрипт динамічно генерує Docker-файл на основі конфігурацій, вказаних у YAML-файлі, включаючи імена модулів, версії та конфігурації портів.

### Структура файлу "buildContainerfile.yml"

Структура файлу "buildContainerfile.yml" складається з різних розділів:

1. Версія: Вказує на версію конфігураційного файлу YAML.
2. Fledge: Містить детальну інформацію про додаток Fledge, включаючи його версію, тип випуску, операційну систему, архітектуру, версію служби диспетчера та версію служби сповіщень.
3. Модулі збірки: Перелічує модулі, які будуть включені в процес збірки. Кожен модуль має назву та версію. Перелічені модулі включають скрипти для побудови різних компонентів або функціональних можливостей, пов'язаних з Fledge.
4. Порти: Дозволяє вказати список портів, які буде відкрито. Ці порти, ймовірно, використовуються програмою Fledge або її модулями для зв'язку або мережевих цілей.
   Ця структура впорядковує інформацію про конфігурацію, необхідну для ефективного створення образу Docker.

### Збірка модулів

Кожен модуль збирається за допомогою спеціального скрипту оболонки, названого за назвою модуля або плагіна, де бажана версія передається як параметр скрипту в Docker-файлі.

### Створення екземплярів модулів та налаштування параметрів Fledge

Скрипт (`importModules.sh`) полегшує інстанціювання потрібних модулів і встановлює основні параметри для Fledge.
